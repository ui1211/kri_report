# 2026-04-22 点群データ OSS 調査

## 要約
本レポートは、点群データの可視化、処理、画像やメタデータ連携に使える OSS を A4 1 枚相当に整理したものである。調査対象は Open3D、CloudCompare、Potree、PCL を中心に、用途別の補助候補として OpenDRIVE 系ツールも含めた。結論として、Python で解析や AI 連携まで見据えるなら Open3D、現場運用や大規模点群の手動確認なら CloudCompare、Web 配信やブラウザ閲覧を重視するなら Potree が有力である。単一ツールで全要件を満たすよりも、解析系と閲覧系を分けて選定する方が現実的である。

## 現状
- 点群 OSS は、解析ライブラリ、デスクトップ可視化、Web ビューアの 3 系統に大別できる。
- Open3D は Python/C++ 両対応で、点群読み込み、幾何処理、描画、機械学習連携までを一通り備える。
- CloudCompare は測量やインフラ点検での利用実績が多く、大規模点群の比較、編集、確認作業に強い。
- Potree は WebGL ベースでブラウザ閲覧に向き、共有や遠隔確認を前提にした構成を取りやすい。
- PCL は C++ 中心の定番ライブラリだが、依存関係と学習コストが重く、導入ハードルは高めである。

## 課題
- 点群処理では、可視化だけでなくメタデータ保持、画像連携、変換時の情報欠落対策まで考える必要がある。
- 大規模点群では、メモリ消費、GPU 活用、LOD 制御の有無が運用性を大きく左右する。
- Python 主体の検証環境では Open3D が有力だが、C++ ビルドや GPU 環境差分が導入負荷になる。
- CloudCompare は実務向きだが GPL ライセンスの扱いとプラグイン拡張の運用設計が必要である。
- Potree は共有性に優れる一方、WebGL 環境や配信基盤の準備が前提となり、解析用途の中核にはしにくい。

## 解決
- 解析基盤は Open3D を第一候補とし、Python から点群入出力、前処理、可視化、ML 連携までを一元化する。
- 現場確認や手動比較用途には CloudCompare を併用し、大規模データの目視確認や差分確認を補完する。
- 社内共有や閲覧用途には Potree を採用し、Web 配信用のビューアとして役割を分離する。
- メタデータや画像のひも付けは単一 OSS に依存せず、元データ管理側で属性設計を維持する方が安全である。
- したがって推奨構成は、`Open3D + CloudCompare` を基本形とし、ブラウザ共有が必要な案件のみ `Potree` を追加する方式である。

## 効果
- Python チームは Open3D を軸に試作から AI 連携まで進めやすく、検証立ち上がりを早められる。
- 現場部門は CloudCompare により、大容量点群の確認や比較を既存運用に近い形で実施できる。
- 閲覧共有を Potree に切り出すことで、解析環境と配信環境を分離し、構成を単純化できる。
- 単一ツール万能主義を避けることで、ライセンス、性能、運用負荷のトレードオフを管理しやすくなる。
- 結果として、用途別に OSS を組み合わせる方針が、短期検証と将来拡張の両面で最も実務的である。
