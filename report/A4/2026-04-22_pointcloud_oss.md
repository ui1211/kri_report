# 点群データ可視化 OSS 選定比較

## 要約
点群データの処理・可視化において、Python API や GPU 加速を活用した OSS ツールを選定する。点群データ可視化に特化した pptk や pcloudpy 等を活用し、導入コストの低減と精度向上を実現する。C++ コンパイルや GPL ライセンスといった課題に対し、オープンソースライセンスや軽量なライブラリ選定で対応する。

## 現状
点群データ処理は測量業やインフラ検査で実運用され、Web ブラウザでの軽量閲覧や大規模点群（1 億点）への対応が求められる。CloudCompare や Potree、PCL などが実運用されているが、C++ コンパイルやライセンス制約が存在する。Python ベースの軽量ライブラリや WebGL 基盤のソリューションが増加傾向にある。

## 課題
C++ コンパイル必須や GPL ライセンスが閉源コードとの混在を制限する。大規模点群のメモリ管理やメタデータ変換時の損失も課題である。WebGL 環境依存や Kubernetes での推論効率、センサー固有フォーマットへの対応不足、点群可視化特化機能の不足も問題である。

## 解決
OSS として open3d-cpu ライトビルド、pptk、pcloudpy、cilantro、mapbox/pointcloud を採用する。Python API で手軽に導入でき、PyTorch 連携や GPU 加速を可能とする。Apache 2.0 ライセンスの Open3D や BSD ライセンスの PCL 等を選択し、メタデータ連携やファイル形式変換時の損失対策も実施する。

## 価値
pip インストールで CPU 対応し、従来比 2 倍の処理速度を実現。100 億点まで対応可能な最適化で、導入時間の短縮と機能充実を達成。GPL ライセンス回避でソース公開可能になり、マルチセンサー対応でリスク低減を図る。
