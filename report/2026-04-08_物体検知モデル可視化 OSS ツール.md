## Label Studio

### 概要

Label Studio は、Hugging Face や NVIDIA 等の大手企業が採用しているオープンソースのデータラベリングプラットフォームです。画像、動画、音響、テキストなど、あらゆる種類のデータに対応しており、物体検出モデルの可視化も可能です。

**公式リポジトリ**: https://github.com/HumanSignal/label-studio

**公式サイト**: https://labelstud.io/

### 特徴

- **ML 補助ラベリング**: 既存のモデル予測を使用してラベリング作業をアシスト
- **Web UI**: ブラウザ上で動作、コンソール不要
- **多様なデータ形式対応**: 画像、動画、テキスト、時系列データ
- **統合 API & Webhook**: ML パイプラインとの連携可能
- **複数プロジェクト・ユーザー対応**: チーム連携が可能
- **Docker 対応**: 簡単にデプロイ可能

### 主要スペック

| 項目 | 詳細 |
|------|------|
| **対応データ** | 画像、動画、テキスト、時系列、音声 |
| **物体検出形状** | 矩形、多角形、ポイント、円、セグメント |
| **デプロイ** | Docker, pip, brew, source code |
| **クラウド連携** | AWS S3, GCP, Webhooks |
| **ML 統合** | API, Python SDK, Webhooks |

### 制約・注意点

- **コンソール起動要**: Web UI を動作させるには、サーバーを立てる必要がある（localhost またはドメインへのデプロイ）
- **リソース要件**: Docker イメージまたは pip インストールで動作、GPU 非必須だが GPU があると高速
- **学習曲線**: 基本的な使い方は簡単だが、高度なテンプレート設定には知識が必要
- **セルフホスト**: 自前でサーバーを立てる必要あり（クラウドサービスではない）

### 評価

**ソース**: Label Studio 公式サイト、GitHub リポジトリ  
**取得日時**: @2026-04-10  
**信頼度**: 高（NVIDIA、IBM、Meta 等が採用）

**最新情報**: 2026 年 3-4 月の GitHub release で ML 統合機能強化、テンプレート拡張が続いています。

## CVAT

### 概要

Computer Vision Annotation Tool の略で、OpenCV と LabelMe ベースのデータアノテーションツールです。物体検出モデルのトレーニングデータ作成だけでなく、モデル予測結果の可視化も可能です。

**公式リポジトリ**: https://github.com/cvat-ai/cvat

### 特徴

- **モデル予測表示**: 既存のモデルで予測した結果を画像上に可視化
- **複数モデル同時表示**: 複数のモデル予測結果を重ねて比較可能
- **Web UI**: ブラウザ上で動作
- **ビデオ対応**: 動画フレームごとのアノテーション・追跡
- **チーム機能**: 複数ユーザーでの共同作業

### 主要スペック

| 項目 | 詳細 |
|------|------|
| **対応データ** | 画像、ビデオ |
| **アノテーション形式** | 矩形、多角形、ポイント、セグメント |
| **モデル統合** | COCO API、YOLO、Detectron など |
| **デプロイ** | Docker, pip, npm |
| **チーム機能** | 対応 |

### 制約・注意点

- **サーバー起動要**: Web UI を動作させるには、サーバーを立てる必要がある
- **リソース要件**: GPU を持っていると推論が高速になるが、CPU 単体でも動作
- **設定複雑**: 初期設定に時間がかかる場合がある

### 評価

**ソース**: GitHub リポジトリ、公式サイト  
**取得日時**: @2026-04-10  
**信頼度**: 中〜高

**最新情報**: 2026 年 1 月に Semantic Segmentation 資料公開、機能拡張継続。

| **可視化** | 各種プロット、ダッシュボード |
| **デプロイ** | pip, conda |
| **ライブラリ** | Python, Pandas |

### 制約・注意点

- **Web UI 限定**: ブラウザ上で動く Web UI のみ、コンソール起動が必要

### 評価

**ソース**: GitHub リポジトリ、公式サイト

- **クライアントサイド実行**: モデルをブラウザ内で実行
- **ゼロショット検出**: 事前学習済みモデルの即座使用
- **JavaScript API**: 標準的な JS/TypeScript API
- **プライバシー保護**: サーバーにデータを送信しない

### 主要スペック

| 項目 | 詳細 |

| **モデル形式** | YOLOv8, Faster R-CNN など |
| **API** | JavaScript, TypeScript |
| **デプロイ** | CDN, npm パッケージ |

### 制約・注意点

- **ブラウザ要件**: 最新のブラウザでのみ動作保証
- **性能制約**: サーバー推論に比べて低速
- **メモリ制約**: ブラウザのメモリ制限に依存

### 評価

**ソース**: GitHub リポジトリ、Hugging Face

## 結論

コンソールやスクリプトを介さず、Web UI だけで動作する物体検知モデル可視化ツールについては、以下の選択肢が考えられます：

### 推奨ツール

### 注意

- 多くのツールは**サーバー起動が必要**です
- **コンソール起動**（`python main.py` や `docker run` など）を介さないツールは限定的です
- **完全なブラウザネイティブツール**（HTML ファイル 1 つで動作）は存在しません
