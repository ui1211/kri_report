## 現状

### 事例

**Label Studio** - サイテキ AI によると、Hugging Face や NVIDIA のような大手企業が採用しているデータラベリングプラットフォームです。画像、動画、音響、テキストなどあらゆる種類のデータに対応しており、物体検出モデルの可視化も可能です。<a href="https://saiteki-ai.com/basics/ai-tool/label-studio/">[Label Studio - サイテキ AI 徹底解説](https://saiteki-ai.com/basics/ai-tool/label-studio/)</a>

### 点検方法

- **Web UI ベース**: Label Studio、CVAT はブラウザ上で動作（サーバー起動が必要）
- **API ベース**: MediaPipe、OpenCV-python は API として使用（サーバー起動は別）
- **ブラウザネイティブ**: Transformers.js はブラウザ内で直接実行

## 課題

### AI 精度

- ブラウザ実行ツール（Transformers.js）はサーバー推論に比べて低速
- ブラウザのメモリ制限に依存
- CNN 活性化マップや Transformer 構造の可視化には特定の技術スタックが必要

### 人不足・運用面

- 多くのツールでサーバー起動が必要（localhost またはドメインへのデプロイ）
- Python 依存が多い（NiceGUI、Dash、Streamlit など）
- 高度なテンプレート設定には知識が必要（Label Studio など）

### OSS

### 研究・実運用事例

- サイテキ AI が Label Studio 導入事例を紹介

- Qiita で Streamlit vs Dash 比較記事が多数

### 価値

- **コスト削減**: Open-source で無料利用可能
- **精度向上**: 最新のバージョンで機能追加続々（例：CVAT の HuggingFace 統合）
- **自動化**: 自動アノテーション機能（Label Studio、CVAT）
- **リスク低減**: 複数ツールで冗長性確保可能

## Label Studio

### 概要

Label Studio は、Hugging Face や NVIDIA のような大手企業が採用しているオープンソースのデータラベリングプラットフォームです。画像、動画、音響、テキストなどあらゆる種類のデータに対応しており、物体検出モデルの可視化も可能です。<a href="https://saiteki-ai.com/basics/ai-tool/label-studio/">[Label Studio - サイテキ AI 徹底解説](https://saiteki-ai.com/basics/ai-tool/label-studio/)</a>

**公式サイト**: https://labelstud.io/

### 特徴

- **ML 補助ラベリング**: 既存のモデル予測を使用してラベリング作業をアシスト
- **Web UI**: ブラウザ上で動作、コンソール不要（サーバー起動が必要）
- **多様なデータ形式対応**: 画像、動画、テキスト、時系列データ
- **統合 API & Webhook**: ML パイプラインとの連携可能
- **複数プロジェクト・ユーザー対応**: チーム連携が可能
- **Docker 対応**: 簡単にデプロイ可能

### 主要スペック

| 項目 | 詳細 |
|---|---|
| **対応データ** | 画像、動画、テキスト、時系列、音声 |
| **物体検出形状** | 矩形、多角形、ポイント、円、セグメント |
| **デプロイ** | Docker, pip, brew, source code |
| **クラウド連携** | AWS S3, GCP, Webhooks |
| **ML 統合** | API, Python SDK, Webhooks |

### 制約・注意点

- **コンソール起動要**: Web UI を動作させるには、サーバーを立てる必要がある（localhost またはドメインへのデプロイ）
- **リソース要件**: Docker イメージまたは pip インストールで動作、GPU 非必須だが GPU があると高速
- **設定複雑**: 基本的な使い方は簡単だが、高度なテンプレート設定には知識が必要
- **セルフホスト**: 自前でサーバーを立てる必要あり（クラウドサービスではない）

### 評価

**ソース**: サイテキ AI、CVAT Blog、GitHub リポジトリ  
**取得日時**: 2026-04-10  
**信頼度**: 高（NVIDIA、IBM、Meta 等が採用、2025 年 9 月更新の信頼情報）

## CVAT

### 概要

Computer Vision Annotation Tool の略で、OpenCV と LabelMe ベースのデータアノテーションツールです。物体検出モデルのトレーニングデータ作成だけでなく、モデル予測結果の可視化も可能です。<a href="https://www.cvat.ai/resources/blog/cvat-or-label-studio-which-one-to-choose/">[CVAT Blog: CVAT vs Label Studio 比較](https://www.cvat.ai/resources/blog/cvat-or-label-studio-which-one-to-choose/)</a>

**公式サイト**: https://www.cvat.ai/

### 特徴

- **モデル予測表示**: 既存のモデルで予測した結果を画像上に可視化
- **複数モデル同時表示**: 複数のモデル予測結果を重ねて比較可能
- **Web UI**: ブラウザ上で動作
- **ビデオ対応**: 動画フレームごとのアノテーション・追跡
- **チーム機能**: 複数ユーザーでの共同作業
- **HuggingFace 統合**: 2025 年に HuggingFace Transformers モデルライブラリを自動アノテーションに統合（CVAT Blog 発表）

### 主要スペック

| 項目 | 詳細 |
|---|---|
| **対応データ** | 画像、ビデオ |
| **アノテーション形式** | 矩形、多角形、ポイント、セグメント |
| **モデル統合** | COCO API、YOLO、Detectron、HuggingFace |
| **デプロイ** | Docker, pip, npm |
| **チーム機能** | 対応 |

### 制約・注意点

- **サーバー起動要**: Web UI を動作させるには、サーバーを立てる必要がある
- **リソース要件**: GPU を持っていると推論が高速になるが、CPU 単体でも動作
- **設定複雑**: 初期設定に時間がかかる場合がある

### 評価

**ソース**: CVAT Blog, GitHub リポジトリ、Sider AI 比較記事  
**取得日時**: 2026-04-10  
**信頼度**: 高（CVAT 公式ブログからの情報、2025 年の機能追加反映済み）

## 結論

画像解析の可視化用 OSS ツールについては、以下の選択肢が考えられます：

### 推奨ツール

1. **Label Studio**: 機能性、柔軟性、ML 統合のバランスが良い
2. **Streamlit**: Python コードでの簡易 UI 生成、共有容易
3. **Transformers.js**: ブラウザ内での直接実行、プライバシー保護
4. **Mediapipe**: リアルタイム画像解析処理
5. **OpenCV-python**: 画像処理基本機能

### 用途別推奨

- **データアノテーション**: Label Studio、CVAT
- **モデル監視**: Evidently AI、Label Studio
- **リアルタイム可視化**: Streamlit、MediaPipe
- **グラフ描画**: Matplotlib、Plotly
- **クラウド共有**: Voiceworks

### 注意

- 多くのツールは**サーバー起動が必要**です
- **コンソール起動**（`python main.py` や `docker run` など）を介さないツールは限定的です
- **完全なブラウザネイティブツール**（HTML ファイル 1 つで動作）は存在しません
- **Python 依存**が多いので、環境構築には時間がかかる場合がある
