# スキル標準化計画書

## 1. 概要

本計画は、`skills/` 配下のフォーマットスキルを標準化し、弱い LLM でも制御が正しく働く汎用的なスキルに昇華させることを目的とする。

## 2. 現状分析

### 2.1 スキル一覧

| スキル | 現在の状態 | 課題 |
|--------|----------|---|
| report-format | 6 セクション固定 | 柔軟性不足 |
| report-format-kri | 10 セクション | 複雑、弱い LLM での制御困難 |
| report-format-a4 | 5 セクション固定 | 文字数制限厳守困難 |
| report-fact-check | 指示のみ | 実装なし |
| research-report | 旧ワークフロー | report-format-kri に統合必要 |

### 2.2 レポート分析結果

| ファイル | report-format-kri 準拠 | 備考 |
|---------|---------|-|
| 2026-04-23_画像認識 AI 異常検知 OSS 事例 | 準拠なし | テンプレート未適用 |
| 2026-04-22_pointcloud_oss | 準拠あり | テンプレート適用済み |

### 2.3 良い点・悪い点

#### 良い点
- 最新のレポート（2026-04-xx）では report-format-kri テンプレートが適用
- マクロ（業界全体）とミクロ（個別トピック）の分離が明確
- C 数値（課題 ID）と解決の対応関係が維持
- 本文リンクが適切に配置

#### 悪い点
- 古いレポートはテンプレート未適用
- 一部のレポートでセクション順序が固定化されていない
- リンク不足や「不明」の記載が散見

### 2.4 弱 LLM で制御が正しく働かない原因

1. **report-format の 6 セクション固定**
   - 柔軟性がない、弱い LLM が「追加するべきセクション」を判断できない

2. **report-format-kri の 10 セクション複雑**
   - 弱い LLM で「C1, C2...」の ID 管理が困難
   - 因果関係の「現状 → 課題 → 解決 → 価値」が崩れやすい

3. **report-fact-check の実装なし**
   - 指示のみで弱い LLM で事実検証を行えない

4. **research-report のワークフロー不整合**
   - report-format-kri との連携不足

## 3. 標準化ルール

### 3.1 必須項目

- **10 セクションの構造維持**
- **「現状 → 課題 → AI 解決 → 価値」の流れ**
- **本文リンク必須**
- **課題 ID（C1, C2...）付与**
- **価値の定量性（数値・比較）**

### 3.2 禁止事項

- 構造の変更
- 注釈・脚注の使用
- 推測（「不明」と書く）

### 3.3 スキル関係図

```
research-report (調査ワークフロー)
  ↓
report-format-kri (報告フォーマット)
  ↓
  ┌─────────────────────┬───────────────┐
  │                     │               │
report-format-a4     report-format   report-fact-check
(要約 A4)        (統合済み)        (品質検証)
```

## 4. 更新手順

### 4.1 report-format-kri の更新

- 構造を 10 セクションに固定
- 「現状 → 課題 → AI 解決 → 価値」の流れを強調
- 本文リンク必須を明確化

### 4.2 research-report の更新

- report-format-kri を使用したワークフロー定義
- 事前分岐：新規作成 vs 更新
- 調査プロセス：検索 → 評価 → ファクトチェック
- レポート生成：report-format-kri テンプレート適用

### 4.3 report-format の統合

- report-format-kri が report-format のスーパーセットとして機能
- 6 セクション固定の機能を report-format-kri に含める

### 4.4 report-format-a4 の維持

- A4 要約用として独立させる
- 文字数制限は「要約のみ」に適用

### 4.5 report-fact-check の実装

- 事実検証のチェックリストを作成
- 検証結果の出力形式を定義
